Executive Summary

This Terraform module deploys Azure Storage Accounts with full support for all four storage services: blob containers, file shares, queues, and tables. It includes lifecycle management policies for automated data tiering and deletion, blob versioning and change feed, static website hosting, immutability policies for compliance, network rules with IP/VNet/private link access controls, private endpoints, CMK encryption, infrastructure encryption, managed identity, CORS rules, and diagnostic settings. The module provides a complete data storage foundation for cloud-native applications.

Component Breakdown

Storage Account

Core account with configurable tier (Standard/Premium), replication (LRS through RAGZRS), kind (StorageV2, BlobStorage, BlockBlobStorage, FileStorage), access tier (Hot/Cool), HNS for Data Lake Gen2, NFSv3, large file shares, infrastructure encryption, and CMK support.

Blob Containers

Container storage with access type control (private/blob/container). Blob properties enable versioning, change feed, last access time tracking, delete retention policies, container delete retention, and CORS rules.

File Shares

SMB/NFS file shares with configurable quota, access tier (TransactionOptimized/Hot/Cool), protocol selection, metadata, and ACL-based access policies with permissions, start, and expiry dates.

Queues and Tables

Storage queues with metadata support. Storage tables with ACL-based access policies for fine-grained access control.

Lifecycle Management

Automated data management with rules for base blobs (tier to cool/archive, delete), snapshots, and versions. Filters by prefix and blob index tags.

Security and Network

Network rules with default deny, Azure Services bypass, IP rules, VNet subnet IDs, and private link access. Private endpoints with DNS zone groups. Immutability policies for WORM compliance.

Data Flow

  1. Clients access storage via public endpoints (filtered by network rules) or private endpoints
  2. Authentication via shared access keys, SAS tokens, or managed identity
  3. Data is stored in appropriate service: blobs, files, queues, or tables
  4. Blob versioning and change feed track data mutations
  5. Lifecycle management policies automatically tier or delete data based on age
  6. Delete retention policies enable soft-delete recovery
  7. Immutability policies enforce WORM storage for compliance
  8. Static website hosting serves web content directly from blob storage
  9. Diagnostic settings stream access logs and metrics to monitoring targets

Industry Adaptation

Data Lake / Analytics

Enable HNS for Data Lake Gen2, configure lifecycle policies for data tiering, use versioning for data lineage, enable change feed for CDC pipelines.

Compliance / Legal

Immutability policies for WORM compliance, CMK encryption, disabled public access, private endpoints, long retention policies, full audit logging.

Web Applications

Static website hosting with custom error pages, CORS rules for cross-origin access, CDN integration, blob containers for media storage.

Backup / Archive

Cool/Archive access tiers, lifecycle management for automated archival, GRS/RAGRS replication for geo-redundancy, immutability for backup integrity.

Production Readiness Checklist

  • Select appropriate replication type (GRS/RAGRS for production)
  • Set minimum TLS version to 1.2 and enable HTTPS-only
  • Disable public blob access (allow_nested_items_to_be_public = false)
  • Configure network rules with default deny action
  • Deploy private endpoints for secure connectivity
  • Enable soft delete for blobs and containers
  • Configure lifecycle management policies for cost optimization
  • Enable blob versioning for data protection
  • Consider disabling shared access keys for AAD-only auth
  • Enable CMK encryption for compliance requirements
  • Configure diagnostic settings for access monitoring
  • Apply resource tags for governance and cost tracking
